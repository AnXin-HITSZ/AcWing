## 一、创建菜单页面

我们先分析一下页面布局。

每个页面都会包括一个导航栏区域和内容区域。导航栏（Navbar）是不变的（除了聚焦显示功能），而变化的只有内容（Content）区域。

因此，我们可以为导航栏创建一个组件。

同时，每个页面也需要分别单独写一个组件。需要包括以下页面：
1. PK 页面；
2. Record 页面；
3. Ranklist 页面；
4. User Bots 页面；
5. 404 页面。

bootstrap 地址：[bootstrap 地址](https://v5.bootcss.com/)。

> 注意：bootstrap 的功能可以简单概括为**方便程序员构建美观的页面**。

## 二、创建游戏页面

我们首先实现游戏地图。

游戏地图设计目标如下：为了利于优势者，地图的尺寸需要设计为奇数长乘偶数长，因此地图尺寸被设计为 $13 \times 14$；由于两方蛇开始时分别位于地图的左下角和右上角，因此为了公平起见地图需设计为中心对称；地图的最外围一圈为障碍物构成的墙壁，地图内的障碍物随机分布且可手动更改障碍物密度；每次刷新页面后地图中障碍物的布局均会随机刷新；合法的地图被定义为从左下角到右上角需为连通的。

游戏地图创建完成后，接下来实现蛇类的建模及键盘的输入操作。

双方的蛇初始时分别位于地图的左下角和右上角。为了增强游戏的趣味性，将随时间动态改变双方的蛇的身体长度；我们人为规定：前十步，每一步身体将变长 1；之后的每 3 步身体将变长 1。

为了实现蛇类的连贯移动，我们的策略为：在移动时，只有头部和尾部移动，而中间的部分保持不动；处理头部的移动时，创建一个虚拟的头部，控制该虚拟的头部向终点方向移动，而尾部始终向终点方向移动。

我们目前无法从后端获取用户的操作，因此只能从前端获取。我们规定玩家 1 的操作键分别为 w、s、a、d，玩家 2 的操作键分别为 ↑、↓、←、→。
